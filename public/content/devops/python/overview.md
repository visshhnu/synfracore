## What is Python for DevOps?

Python is the #1 scripting language for DevOps automation. You need to write clean automation scripts, not be a full developer. Focus: server health checks, deployment scripts, API calls to Kubernetes/AWS/Azure, log processing.

## Why Python for DevOps?

Dictionaries are the most important data type for DevOps — they map directly to JSON responses from Kubernetes API, AWS, Azure. Master dict comprehensions and nested dict traversal.

## Learning Modules

### Module 01 — Why Python for DevOps
*Use cases and the DevOps mindset*

Python is the #1 scripting language for DevOps automation. You need to write clean automation scripts, not be a full developer. Focus: server health checks, deployment scripts, API calls to Kubernetes/AWS/Azure, log processing.

**Topics covered:**

- Infrastructure automation scripts — 🟢 Beginner
- REST API integrations — 🟢 Beginner
- Log parsing and alerting — 🟢 Beginner
- Cloud SDK automation — 🟡 Intermediate

### Module 02 — Variables & Data Types
*str, int, list, dict — DevOps focus*

Dictionaries are the most important data type for DevOps — they map directly to JSON responses from Kubernetes API, AWS, Azure. Master dict comprehensions and nested dict traversal.

**Topics covered:**

- F-strings and string formatting — 🟢 Beginner
- Lists and iteration — 🟢 Beginner
- Dictionaries (JSON mapping) — 🟢 Beginner
- Type checking — 🟢 Beginner

### Module 03 — Functions & Error Handling
*Write reusable automation*

DevOps scripts must handle failures gracefully. Always wrap API calls in try/except. Retry logic is essential for network operations. Use specific exception types, not bare except.

**Topics covered:**

- Function definition and args — 🟢 Beginner
- try/except/finally — 🟡 Intermediate
- Custom exceptions — 🟡 Intermediate
- Retry logic — 🟡 Intermediate

### Module 04 — File & OS Operations
*Read logs, run commands, manage files*

Scripts read config files, parse logs, and run shell commands. subprocess is the modern way to run commands. Always use pathlib over os.path for file paths. Parse YAML/JSON directly — no regex on structured data.

**Topics covered:**

- File read/write operations — 🟢 Beginner
- subprocess module — 🟡 Intermediate
- os and pathlib — 🟡 Intermediate
- Parsing JSON and YAML — 🟡 Intermediate

### Module 05 — REST APIs & HTTP
*Kubernetes API, cloud APIs*

The requests library is your primary HTTP tool. Kubernetes API is REST — kubectl is a client over it. boto3 is AWS SDK, azure-sdk for Azure. Always handle pagination and rate limiting.

**Topics covered:**

- requests library — 🟢 Beginner
- Authentication (Bearer, API keys) — 🟡 Intermediate
- Kubernetes API calls — 🟡 Intermediate
- boto3 / azure-sdk — 🔴 Advanced
