# Git — Prerequisites

## Git is Beginner-Friendly

Git is the first DevOps tool most people learn. The prerequisites are minimal.

## Required

### 1. Basic Computer Skills
- Can create folders and files
- Understand file paths (`/home/user/projects/my-app`)
- Can open a terminal/command prompt

### 2. A Text Editor
You'll edit code files and write commit messages. Any of these work:
- **VS Code** (recommended — free, popular, great Git integration)
- **Sublime Text**
- **nano** (built into Linux/Mac — simplest)
- **vim** (powerful but steep learning curve — learn later)

### 3. A GitHub Account
Almost all Git workflows involve GitHub (or GitLab/Bitbucket). Create a free account at **github.com** before starting.

## That's It

Git really does have the lowest barrier of any DevOps tool. You don't need Linux knowledge, programming experience, or cloud accounts to start.

## What You'll Build Toward

After Git basics, the natural progression is:
```
Git basics (this section) ← Start here
      ↓
GitHub collaboration (branching, PRs, code review)
      ↓
CI/CD (GitHub Actions triggers on your Git pushes)
      ↓
GitOps (ArgoCD watches your Git repo to deploy to Kubernetes)
```

Git is the foundation everything else builds on. Invest time getting it solid.

## Time Estimate

- Basic Git workflow (add, commit, push, pull): 1 day
- Branching and merging: 2-3 days
- Comfortable with Git in a team workflow: 1-2 weeks
