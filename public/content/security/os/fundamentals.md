# Operating Systems Fundamentals

## What is an Operating System?
An OS manages hardware resources and provides services to applications. It acts as an intermediary between users/programs and hardware.

**Key functions:**
- Process management
- Memory management
- File system management
- I/O device management
- Security and access control

## Process vs Thread

### Process
- Independent unit of execution with its own memory space
- Creating processes is expensive (fork/exec)
- Inter-Process Communication (IPC): pipes, sockets, shared memory, signals

### Thread
- Lightweight execution unit within a process
- Shares memory, file handles with parent process
- Cheaper to create, context-switch than processes
- Risks: race conditions, deadlocks

### Process States
New → Ready → Running → Waiting → Terminated

## CPU Scheduling

### Scheduling Criteria
- **CPU Utilization**: Keep CPU busy
- **Throughput**: Processes completed per unit time
- **Turnaround Time**: Total time from submission to completion
- **Waiting Time**: Time spent in ready queue
- **Response Time**: Time from request to first response

### Scheduling Algorithms
- **FCFS**: First Come First Served. Simple but convoy effect.
- **SJF**: Shortest Job First. Optimal for average wait time. Requires burst time prediction.
- **Priority Scheduling**: Higher priority runs first. Starvation risk — solved by aging.
- **Round Robin**: Each process gets a time quantum. Fair, good for time-sharing.
- **Multilevel Queue**: Different queues for different process types (interactive, batch).

## Memory Management

### Address Spaces
- **Logical address**: Generated by CPU
- **Physical address**: Actual memory location
- **MMU**: Memory Management Unit translates logical to physical

### Paging
Memory divided into fixed-size pages (logical) and frames (physical).
- **Page Table**: Maps pages to frames
- **TLB**: Translation Lookaside Buffer caches recent translations
- **Page fault**: Page not in memory, loaded from disk

### Segmentation
Memory divided into variable-size logical segments (code, stack, heap).

## File Systems

### File System Structure
- **Boot block**: OS loader
- **Super block**: File system metadata
- **Inode table**: File metadata (permissions, size, timestamps, block pointers)
- **Data blocks**: Actual file content

### Directory Structure
- Single-level, two-level, tree-structured, acyclic graph

### File Allocation Methods
- **Contiguous**: Fast access, external fragmentation
- **Linked**: No fragmentation, no random access
- **Indexed**: Index block points to data blocks, random access supported

## Deadlocks

### Conditions (all 4 must hold)
1. **Mutual Exclusion**: Only one process uses resource at a time
2. **Hold and Wait**: Process holds resource while waiting for another
3. **No Preemption**: Resources not forcibly taken
4. **Circular Wait**: P1 waits for P2 waits for P3 waits for P1

### Handling Deadlocks
- **Prevention**: Eliminate one of the four conditions
- **Avoidance**: Banker's Algorithm — don't enter unsafe state
- **Detection and Recovery**: Let deadlock happen, detect, recover
- **Ignore**: Ostrich algorithm (used in many OS)
