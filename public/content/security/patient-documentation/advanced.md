# Patient Documentation Advanced

## Clinical Documentation Improvement (CDI)

### CDI Program Goals
- Ensure documentation accurately reflects patient severity
- Improve DRG and HCC capture
- Reduce queries (proactive education of physicians)
- Support quality measures and public reporting

### CDI Specialist Role
- Concurrent review of records during admission
- Query physicians for clarification
- Education on documentation best practices
- Collaboration with coding department

### ROI of CDI Programs
- Case Mix Index (CMI) improvement
- DRG optimization
- Reduced coding queries
- Better quality metrics (risk-adjusted outcomes)

## HCC (Hierarchical Condition Category) Coding

### Purpose
HCC coding used for risk adjustment in Medicare Advantage and ACO models.
Predicts next year's expected costs based on diagnoses.

### Key Principles
- All chronic conditions must be coded every year (cannot rely on prior year)
- Specificity matters (Type 2 DM with CKD stage 3 > Type 2 DM)
- Provider must document and code all conditions actively managed

### High-Value HCC Conditions
- Diabetic complications
- Heart failure
- CKD stages 3-5
- COPD
- Rheumatoid arthritis
- Major depression
- Polyneuropathy

## Legal Aspects of Documentation

### Medical Records as Legal Documents
- Records are admissible evidence in malpractice cases
- Late entries must be identified as such with current date
- Do not alter records; use corrections with single line, initial, date
- Never destroy records in anticipation of litigation

### Informed Consent Documentation
- What procedure/treatment was explained
- Risks and benefits discussed
- Alternatives presented
- Patient questions answered
- Patient signed consent or reason consent waived

## Interoperability and EHR Documentation

### FHIR (Fast Healthcare Interoperability Resources)
- HL7 standard for exchanging healthcare information electronically
- RESTful API-based
- Enables seamless data sharing between EHR systems

### Structured vs Unstructured Data
- Structured: coded fields (diagnoses, meds) — machine-readable
- Unstructured: free text notes — requires NLP to extract
- EHR design affects coding accuracy and completeness
