# Ethical Hacking — Interview Questions

**What is the methodology of a penetration test?**
Reconnaissance: passive (OSINT — gather public info without touching target) then active (port scanning, service enumeration). Scanning and vulnerability assessment: identify open ports, services, OS versions, known CVEs. Exploitation: attempt to exploit found vulnerabilities using tools like Metasploit, custom exploits, or manual techniques. Post-exploitation: privilege escalation, lateral movement, persistence, data exfiltration simulation. Reporting: document every finding with evidence, CVSS severity rating, business impact, and remediation recommendations. Rules of engagement define scope — what's in/out of bounds. Always get written authorization first.

**What is the difference between a vulnerability scan and a penetration test?**
Vulnerability scanning is automated — tools (Nessus, OpenVAS, Qualys) scan for known CVEs, misconfigured services, and weak credentials. Fast, covers broad surface, but generates false positives and doesn't chain vulnerabilities. Penetration testing is manual, adversarial simulation — a human tester attempts to actually exploit vulnerabilities, chain multiple weaknesses to achieve impact (like combining an SSRF with IAM misconfiguration to gain cloud access), and tests business logic flaws that scanners miss. Pen tests are slower and more expensive but find what matters. Run vulnerability scans continuously; pen tests annually or after major changes.

**Explain SQL injection types.**
Classic (in-band error-based): `' OR '1'='1` — error messages reveal database structure. Union-based: append `UNION SELECT` to retrieve data from other tables. Blind (boolean-based): no output visible, but behavior changes (login success/fail) reveals information one bit at a time. Time-based blind: `'; WAITFOR DELAY '0:0:5'--` — database delays confirm condition is true, extract data character by character. Out-of-band: use DNS/HTTP channels to extract data (when other methods are blocked). Second-order: input stored and later used in a query without re-sanitization. Fix: parameterized queries/prepared statements for all cases, no concatenation of user input into SQL.

**What is privilege escalation and what are common vectors in Linux?**
Privilege escalation = gaining higher permissions than initially granted (usually escalating to root). Common vectors: SUID binaries — find with `find / -perm -4000 -type f`; if vim, bash, or python has SUID, trivial root; check GTFOBins.github.io for each binary. Writable cron jobs running as root — inject malicious command. Sudo misconfiguration — `sudo -l` to see what you can run; `sudo vim` → `:!/bin/bash` gives root shell. Weak file permissions — world-writable /etc/passwd or /etc/shadow. Kernel exploits — `uname -r` reveals kernel version; search for unpatched CVEs. NFS misconfiguration — root squash disabled. LinPEAS automates this enumeration.

**What is the OWASP Top 10 and what is the most commonly found vulnerability?**
OWASP Top 10 is the industry standard awareness document for web application security risks, updated every 3-4 years based on data from real applications. The 2021 list: Broken Access Control (jumped to #1), Cryptographic Failures, Injection, Insecure Design, Security Misconfiguration, Vulnerable Components, Auth Failures, Integrity Failures, Logging Failures, SSRF. Broken Access Control (IDOR — Insecure Direct Object References) is the most commonly found vulnerability in real pen tests — applications fail to verify if the authenticated user is authorized to access specific resources. Example: changing `userId=123` to `userId=124` in an API call returns another user's data.
