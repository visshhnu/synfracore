## Three-Pass Strategy
```
Pass 1 (first 40% of time):
  Attempt all EASY questions — instant recognition, <30 sec each
  Skip medium/hard — mark for later

Pass 2 (next 40% of time):
  Attempt MEDIUM questions — willing to spend up to 2× average time
  Use partial knowledge to eliminate options

Pass 3 (final 20% of time):
  Attempt HARD questions if time permits
  Apply negative marking calculation before attempting

Final 5-10 min:
  Review flagged answers; ensure all answered (check negative marking)
```

## Negative Marking — When to Attempt
| Marking Scheme | Minimum Confidence to Attempt |
|---------------|-------------------------------|
| +4, −1 | >20% (eliminate at least 0 wrong options) |
| +2, −0.5 | >20% |
| +3, −1 | >25% |
| +2, −2/3 | >25% |
| +1, −1/3 | >25% |
Formula: Attempt if P(correct) > |penalty| / (reward + |penalty|)

## Dangerous Trap Words
| Word | Caution |
|------|---------|
| ALWAYS / NEVER | Usually wrong — exceptions almost always exist |
| ONLY | Often too restrictive |
| ALL | May have important exceptions |
| EXCEPT / NOT | High misread risk — reread question twice |
| None of the above | Legitimate option — don't dismiss |
| All of the above | Verify every option carefully before selecting |

## Error Classification
| Error Type | Root Cause | Corrective Action |
|-----------|-----------|-----------------|
| Conceptual | Topic not understood | Study the concept |
| Careless | Misread or didn't verify | Slow down; reread twice |
| Calculation | Arithmetic mistake | Show working; verify key steps |
| Time pressure | Ran out of time | Speed drills; time-split strategy |
| Trap | Chose the "obvious" wrong answer | Read ALL options every time |

## Post-Mock Analysis — Mandatory Steps
☑ Calculate score and estimated percentile

☑ Classify each error by type (concept/careless/calculation/time/trap)
☑ Update error log with new entries
