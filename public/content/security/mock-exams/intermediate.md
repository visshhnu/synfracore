# Mock Exams — Analysis and Improvement

## Error Classification System

### Four Error Types
1. **Conceptual error**: Didn't know the underlying concept → Study the topic
2. **Careless error**: Knew the concept but made a mistake → Slower, more careful approach
3. **Calculation error**: Arithmetic mistake → Practice calculation speed and accuracy
4. **Time pressure error**: Knew the answer but ran out of time → Speed drills, time management

### Error Log Best Practice
Maintain a running error log (notebook or spreadsheet):
```
Date | Exam | Q# | My Answer | Correct | Error Type | Topic | Root Cause | Revised Understanding
-----|------|-----|-----------|---------|-----------|-------|-----------|---------------------
```
Review error log weekly; look for patterns across multiple mocks.
Most important column: "Root Cause" — what EXACTLY led to the error?

## Subject-Specific Mock Strategy

### Quantitative Aptitude / Mathematics
- One-line solution approach: avoid writing all steps for easy questions
- Estimation: for multiple-choice, narrow to 2 options before exact calculation
- Back-calculation: substitute answer options into problem to verify
- Percentage calculations: use quick fraction equivalents (1/6≈17%, 1/8=12.5%)

### Logical Reasoning
- Series: write differences between consecutive terms; identify second-order differences
- Coding: write the alphabet with positions (A=1, Z=26); find systematic rule
- Blood relations: draw family tree immediately; don't rely on mental model
- Syllogism: Venn diagram method; exhaust all diagram possibilities

### General Knowledge / Current Affairs
- Don't try to memorize everything — understand context of major events
- For static GK: use tables (capitals/currencies/national symbols as linked groups)
- Current affairs: month-wise compilation; last 6 months most tested
- Focus on: government schemes, sports championships, science milestones, appointments

### English / Verbal
- Para jumbles: find the sentence with no pronoun reference (likely opener); find mandatory pair (one sentence refers to another)
- Error spotting: read each option carefully; common errors: subject-verb agreement, pronoun case, tense consistency
- Vocabulary: learn words in context, not isolated lists

## Intensity Schedule

### 60-Day Exam Prep with Mocks
```
Days 1-20: Concept building + topic tests
  2-3 topic tests/week; no full mocks yet

Days 21-45: Mixed topic tests + first full mocks
  Full mock every 5-7 days
  Topic tests for weak areas identified

Days 46-55: Full mock every 2-3 days
  Post-mock analysis same day (1.5-2 hrs)
  Targeted revision based on analysis

Days 56-60: 1 full mock every 2 days
  Focus on consistency and stamina
  Light revision of error log
  Zero new topics
```

## Tracking Progress Across Mocks

### Metrics to Track
- Overall score and percentile (if available)
- Subject-wise accuracy
- Accuracy vs attempt rate (over-attempting vs under-attempting)
- Time per section used
- Trend line across multiple mocks (improving/stable/declining)

If score is declining: check for fatigue, confirm mock conditions are consistent.
If accuracy is declining but attempts increase: you're being reckless — slow down.
If accuracy is fine but score low: increase attempt rate.
