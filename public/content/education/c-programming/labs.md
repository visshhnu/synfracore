# C Programming — Coding Labs

## Interactive Coding Environment

These labs use our code editor with execution simulation and automated test cases.

## Lab Structure

Each lab follows this format:
1. **Concept explanation** — brief review of the topic
2. **Worked example** — see the solution approach
3. **Practice problem** — implement it yourself
4. **Test cases** — verify your solution
5. **Hints** — if you're stuck, reveal progressive hints

## Lab Tracks

### Track 1: Fundamentals
- Hello World and basic I/O
- Variables, data types, type casting
- Operators and expressions
- Conditional statements (if/else, switch)
- Loops (for, while, do-while)
- Functions and recursion

### Track 2: Data Structures
- Arrays: 1D and 2D
- Strings: manipulation and algorithms
- Linked Lists: implementation from scratch
- Stack and Queue implementation
- Binary Search Tree

### Track 3: Algorithms
- Sorting: bubble, selection, insertion, merge, quick sort
- Searching: linear search, binary search
- Recursion patterns: factorial, fibonacci, tower of hanoi
- String algorithms: reverse, palindrome, anagram

### Track 4: OOP (Java/C++)
- Classes and objects
- Inheritance and polymorphism
- Abstract classes and interfaces
- Exception handling

### Track 5: Interview Patterns
- Two pointers technique
- Sliding window
- Frequency counting with HashMap
- Recursion and backtracking
- Dynamic programming basics

## How to Approach Coding Problems

1. **Read carefully** — understand input format, output format, constraints
2. **Examples first** — trace through the examples manually
3. **Brute force** — get any working solution first
4. **Optimize** — then think about time/space complexity
5. **Edge cases** — empty input, single element, maximum values

## Time Complexity Quick Reference
```
O(1)    → HashMap lookup, array access
O(log n) → Binary search, balanced BST operations
O(n)    → Linear scan, single loop
O(n log n) → Merge sort, heap sort
O(n²)   → Bubble sort, nested loops
O(2^n)  → Subset generation, some recursion
O(n!)   → Permutations
```
