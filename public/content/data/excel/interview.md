# Excel & Advanced Excel — Interview Questions

**What is the difference between VLOOKUP and INDEX-MATCH?**
VLOOKUP searches the leftmost column of a range and returns a value from a specified column to the right. Limitations: can only look left-to-right; if you insert a column, column numbers break; slightly slower on large ranges. INDEX-MATCH uses two functions: MATCH finds the position of a value in a range (returns a row number), INDEX returns the value at that position. It can look in any direction, the column reference doesn't break on insertion, and it's faster. In Excel 365/2019+, XLOOKUP replaces both — simpler syntax, searches any direction, returns arrays, has built-in error handling. For new work: use XLOOKUP. For compatibility: INDEX-MATCH is safer than VLOOKUP.

**What is Power Query and why is it better than manual data cleaning?**
Power Query is a data transformation engine built into Excel (Get & Transform). You connect to data sources (CSV, SQL, SharePoint, web), apply transformations through a GUI, and each step is recorded. When data refreshes, all transformations run automatically. Better than manual: reproducible (same transformations every time, no human error); scalable (can handle millions of rows that freeze Excel formulas); maintainable (see every step, modify any step); refreshable (click Refresh and new data processes automatically). Essential for: recurring reports, messy data requiring consistent cleaning, combining multiple files, connecting to databases.

**Explain the difference between absolute and relative cell references.**
Relative reference (`A1`): when you copy the formula, the reference adjusts relative to the formula's new position. Copy `=A1+B1` from row 1 to row 2, it becomes `=A2+B2`. Absolute reference (`$A$1`): the reference is locked and doesn't change when copied. Mixed reference: `$A1` (column locked, row adjusts) or `A$1` (row locked, column adjusts). Practical use: lookup table in a fixed range → use absolute `$A$1:$C$100`. Sum of current row → use relative. Header row that should stay fixed while row increments → use `A$1`. Toggle between types with F4 key when editing a reference.

**What is a Pivot Table and what are its main use cases?**
A Pivot Table dynamically summarizes large datasets — you drag fields to rows, columns, values, and filters to slice data without writing formulas. Main use cases: summarize millions of rows to monthly/quarterly/annual totals; cross-tabulation (region × product × quarter); ranking top N customers/products; quickly testing different groupings (drag-and-drop). Key features: drill-down (double-click a cell to see underlying data), calculated fields, slicers (visual filters), timelines, grouping dates. Best for: exploratory analysis and business reporting. Limitation: doesn't update automatically when source data changes (requires manual refresh) and can't be referenced dynamically in formulas (use GETPIVOTDATA or Power Query instead).

**What is the difference between a Macro and a VBA function?**
A Macro is a recorded sequence of actions that plays back — you record yourself formatting a report, and the macro repeats those exact steps. Useful for repetitive formatting tasks. A VBA (Visual Basic for Applications) function or Sub is written code — you write logic, loops, conditions, user forms, and interact with the Excel object model programmatically. VBA can: loop through 10,000 rows and apply conditional logic; connect to databases; create userforms for data entry; automate cross-workbook operations. VBA is powerful but has limitations: security concerns (macro viruses), no version control, Windows-only, maintenance burden. For new automation: consider Python (openpyxl, pandas) or Power Automate for better maintainability.
